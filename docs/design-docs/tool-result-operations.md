# Tool Result Operations

The coordinated tool-result release owns four deployment invariants:

- AgentK model context is capped at 12,288 bytes.
- The execution-engine active evidence ledger is capped at 49,152 bytes.
- Complete redacted artifacts are capped at 2,097,152 uncompressed bytes.
- Complete redacted artifacts expire after seven days; compact conversation evidence follows the 30-day conversation policy.

Compose and Helm expose these as `TOOL_CONTEXT_MAX_BYTES`, `TOOL_CONTEXT_RUN_MAX_BYTES`, `TOOL_RESULT_ARTIFACT_MAX_BYTES`, `TOOL_RESULT_ARTIFACT_RETENTION_DAYS`, `AGENT_WS_MAX_PAYLOAD_BYTES`, `BUILTIN_MCP_MAX_RESPONSE_BYTES`, and `TOOL_GATEWAY_MAX_RESPONSE_BYTES`. The two AgentK envelope hops default to 3 MiB so a complete result at the 2 MiB artifact ceiling still fits with MCP and routing metadata. The normalized gateway-to-engine hop is bounded at 5 MiB because untrusted results can legitimately appear in both `full_result` and the pre-fallback `model_context`. Service startup rejects transport limits below their supported envelope minimum.

This is a clean contract cutover, not a rolling-version-compatible change. Apply the additive control-plane and gateway migrations first, then stop new run dispatch and drain active runs and pending write approvals before replacing the dependent services. Keep dispatch closed while AgentK, the control plane, gateway, execution engine, console, and deployment configuration converge on the tested release matrix. Force AgentK reconnection and built-in tool resynchronization, verify that every AgentK tool has an output schema and artifact policy, and run read-only contract and Pod ownership checks before reopening writes. Complete the Pod remediation smoke after writes reopen. If the platform cannot provide a drained maintenance window or an equivalent atomic traffic switch, do not release this cutover. Roll back the full matrix while dispatch remains closed if any contract gate fails.

Before release, run the focused Pod-only remediation gate with `PERSIST_RUN_EVENTS=true ACORNOPS_SMOKE_REMEDIATION_ONLY=true ACORNOPS_SMOKE_REMEDIATION_RUNS=20`. The canonical local, staging, and production configurations persist run events; the gate rejects incomplete trace behavior that an in-memory event buffer can hide. The test begins from only a failing Pod identity, requires the exact UID-verified Deployment and container, waits for the built-in agent connection to be live, verifies compact-only events and artifact access, approves the patch, requires a fresh post-patch `get_resource` observation of the repaired image, and checks rollout health. Set `ACORNOPS_SMOKE_REMEDIATION_METRICS_PUSH_URL` to the staging Pushgateway grouping URL so the dedicated gate publishes success and last-run timestamp gauges. Schedule it at least hourly: production alerts when a run fails, either gauge is absent, or the timestamp is more than two hours old. Staging must then soak the coordinated matrix for at least 48 hours.

Prometheus rule assets cover invalid AgentK projections at both gateway and execution-engine validation boundaries, artifact upload SLO, artifact size/authorization anomalies, elevated ownership-resolution failures, failed or missing post-write verification, and failed, missing, or stale synthetic remediation. The chart does not install a Prometheus stack or `PrometheusRule` CRDs; production monitoring must load the files under `observability/prometheus/alerts/`, and staging must verify the rules are active before the soak begins. A contract or remediation gate failure requires rollback of the coordinated matrix rather than an individual service.
